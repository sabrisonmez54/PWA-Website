Progressive Web Apps are Powerful!

Replacing Native Applications

Progressive Web Apps (PWA) are built and enhanced with modern APIs to deliver native-like capabilities, reliability, and installability while reaching anyone, anywhere, on any device with a single codebase. A progressive web application takes advantage of these said APIs to combine the best of web and mobile apps. It is kind of like a website built using HTML, CSS, and Javascript but that acts and feels like an app.

Native applications feel like part of the device they run on whereease traditionally web apps could not provide the same experience. However, this can change due to the capabilities of PWAs.

Sources: web.dev

Capabilities of PWAs

First and foremost, PWAs have a bigger reach web apps are more discoverable than native apps; it's a lot easier and faster to visit a website than to install an application, and you can also share web apps by simply sending a link.

One of the most powerful features of the web is the ability to link to an app at a specific URL without the need for an app store or complex installation process. Another huge advantage of this is an installed PWA usually takes less than 1MB, so say goodbye to storage space issues.

Sources: web.dev, MDN

PWAs are also installable (without an App Store)

A core part of the web app experience is for users to have app icons on their home screen, and be able to tap to open apps into their own native container that feels nicely integrated with the underlying platform.

Installed Progressive Web Apps run in a standalone window instead of a browser tab. They're launchable from on the user's home screen, dock, taskbar, or shelf. It's possible to search for them on a device and jump between them with the app switcher, making them feel like part of the device they're installed on.

Try it yourself! click on the button to add this website to your homescreen

*if you are on iOS, however, you will need to manually press the share icon and then click add to homescreen*

PWAs can work offline

Thanks to service workers and caching PWAs can work when the network is unreliable, or even non-existent. This allows for better speed and realibility. The advantages of network independence allow for:

  • Revisiting of a site and getting its contents even if no network is available.
  • Browsing any kind of content the user has previously visited at least once, even under situations of poor connectivity.
  • Controling of what is shown to the user in situations where there is no connectivity.

    • and all these features can work in any browser!

    Sources: web.dev, MDN

    You can receive Push Messages and show Notifications

    Like with native apps users can be re-engaged by updates and new content, even when they aren't looking at the app or using their devices. Web push notifications allow users to opt-in to timely updates from sites they love and allow you to effectively re-engage them with customized, relevant content

    Try it yourself! click on the button to receive notifications

    Native-like Capabilities

    PWAs can access native device features like the Camera you can build a hyper-local video chat app using WebRTC, geolocation, and push notifications. You can make that app installable and take those conversations virtual with WebGL and WebVR. With the introduction of Web Assembly, developers can tap into other ecosystems, like C, C++, and Rust, and bring decades of work and capabilities to the web too.

    New and upcoming APIs are looking to expand what the web can do with features like file system access, media controls, app badging, and full clipboard support.

    Try it yourself! click on the button to access camera

    They look good on any screen

    PWAs use technologies like media queries and viewport to make sure that their UIs will fit any form factor: desktop, mobile, and tablets.
